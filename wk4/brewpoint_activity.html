Logical AND &&

Both conditions must be true. If even one is false, the whole expression is false.

&&
True only when BOTH sides are true
true&&true→true
true&&false→false
false&&true→false
false&&false→false
Scenario
Free drink redemption
A customer can redeem a free drink only if they are a member AND have at least 50 points.

Part 2 of 4

Logical OR ||

At least one condition must be true. Only fails when both sides are false.

||
True when AT LEAST ONE side is true
true||true→true
true||false→true
false||true→true
false||false→false
Scenario
Happy Hour discount
A customer gets 20% off if it is Happy Hour OR if they are a student.
// Brew Point — Free Drink Check

let isHappyHour = confirm("Is it Happy Hour (2-4pm)? Click OK for Yes, Cancel for No.");

let isStudent = confirm("Is the customer a student? Click OK for Yes, Cancel for No.");

// Use the OR operator: discount if either condition is true

let getsDiscount = isHappyHour || isStudent;

// Explain the logic and show the result

if (getsDiscount) {

alert("The customer gets 20% off!");

} else {

alert("No discount today.");

}
Check Answer

No — both values are false, so the result is false.

Check Answer

Examples:
• isHappyHour = true, isStudent = false
• isHappyHour = false, isStudent = true
• (both true also works)

Check Answer

|| only needs one true value; && requires both to be true.

Part 3 of 4

Logical NOT !

Flips a boolean value. true becomes false, false becomes true.

Scenario
App maintenance check
The app lets a customer place an order only if the system is NOT under maintenance AND they are logged in.

Copy this code into your practice.js file to learn about the ! operator

// Ask the user two questions

let isMaintenance = confirm("Is the system under maintenance? Click OK for Yes, Cancel for No.");

let isLoggedIn = confirm("Is the customer logged in? Click OK for Yes, Cancel for No.");

// The customer can order only if the system is NOT under maintenance AND they are logged in

let canOrder = !isMaintenance && isLoggedIn;

// Explain the logic and show the result

if (canOrder) {

alert("The customer can place an order.");

} else {

alert("Ordering is not available right now.");

}

Check Answer

false — NOT flips true to false.

Check Answer

Step 1: !false → true
Step 2: true && false → false
Final result: false

Check Answer

Using !isMaintenance is shorter and avoids creating an extra variable.

Predict the output

Given this code:

let canGetLateMenu = (isAfter8pm || preOrdered) && !kitchenClosed;

Scenarios:

  1. isAfter8pm = true, preOrdered = false, kitchenClosed = false
  2. isAfter8pm = false, preOrdered = true, kitchenClosed = false
  3. isAfter8pm = false, preOrdered = false, kitchenClosed = false
  4. isAfter8pm = false, preOrdered = true, kitchenClosed = true
Check Answers
  1. true
  2. true
  3. false
  4. true

Evaluate: ! → && → ||. The parentheses around (isAfter8pm || preOrdered) ensure correct order.
